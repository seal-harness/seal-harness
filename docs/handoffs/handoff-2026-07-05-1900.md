## 5. Key Decisions & Rationale (do not undo without reason)

1. **Git replaces the Audited log.** The user explicitly chose: disk is canonical for skills/agents/memory; the Audited log is removed entirely; `~/.seal/config/` is a git repo including `vault.age` (ciphertext — safe; identity in `keys/` off-repo). Model-authored writes auto-commit; humans commit file-drops. This was a deliberate pivot away from the M1 Audited-log design — do not re-introduce an Audited log.
2. **Memory lives in `config/memory/`, in git.** The user chose git-tracking for memory too (it's runtime data the model writes, but the user wanted it version-controlled alongside skills/agents). Memory files are Markdown (frontmatter = tags/created/updated/session, body = content).
3. **YAML-ish frontmatter, hand-rolled.** `---` fences (the universal convention — Jekyll/Hugo/Obsidian/pureclaw), flat `key: value` lines, JSON arrays for list fields. No YAML dependency added (aeson encodes the arrays). pureclaw's `Memory/Markdown.hs` uses the same `---`-fence style.
4. **`Audited` `TrustLevel` constructor kept.** Removing it would cascade into test fixtures and the roadmap's `TrustLevel` enumeration. The dispatcher's `Audited` arm treats it as Trusted (records to session transcript only, no Audited log). All opcodes that were `Audited` are now `Trusted`.
5. **Sub-agents get their own two-file transcript** under `<parent-session>/agents/<child-id>/`. The two-file format's `erConvLen` and envelope-delta fold are per-session; sharing the parent's `TwoFileHandle` would corrupt reconstruction. The sub-agent shares the parent's AuditedHandle... **except the AuditedHandle no longer exists** (deleted in the pivot). The sub-agent now shares nothing cross-session except the config git repo (disk canonical). The `mkWorker` in `Cli.hs` opens a fresh `withTwoFileTranscript childDir` per AGENT_START.
6. **`/agent` is discovery-only** (mirrors pureclaw): `/agent list|info|default`. The running-agent lifecycle (`AGENT_START/STOP/STATUS`) is reachable only via the ISA tool calls, not slash commands. Skills have no `default` (on-demand invocation only): `/skill list|info`.
7. **Commit messages: Write a clear message describing the *why*; one commit per task; never `--no-verify`.
