## 5. Key Decisions & Rationale (do not undo without reason)

1. **Two append-only logs, distinct roles.** Session transcript = per-session record, new two-file format (`conversation.jsonl` + `entries.jsonl`), **disposable** (user may prune old sessions). Audited log = single global cross-session record of mutations to the four evolutionary stores, **canonical** for those stores, **not prunable**, off-box mirror hook. Audited opcodes write to BOTH. Deleting a session transcript loses the conversation but no evolutionary state.
2. **No hash chain.** Integrity rests on the single-writer + fsync + keeping untrusted ops off the box that holds the log — same foundation as the session transcript. A hash chain without proof-of-work mining adds no real guarantee. `Seal.Audited.Chain.verifyOrder` is a basic sanity check (unique ids), not tamper-evidence.
3. **`CbToolResult` parts are redacted** in `conversation.jsonl` (replaced with `<redacted:secret>`) so secret values returned by `SECRET_GET` never reach disk. The agent still sees the real value in-memory. (User chose redaction over skipping.)
4. **Audited log payload = opcode INPUT**, not `orRecorded`. The dispatcher writes before `opRun` returns, so it can't write `orRecorded`. The materializer reconstructs full entries (id/content/tags from input + timestamps/session from the Audited event's `aeEvTs`/`aeEvSession`). This keeps the dispatcher uniform across Audited opcodes.
5. **Memory content is agent-visible data, recorded in full** in both logs. It is NOT a vault secret (those live in `Seal.Security.Secrets`). The "no secret ever serialized" rule applies to vault secrets, not memory content.
6. **`noneBackend` (in-memory) shipped for M2.** SQLite and Markdown backends deferred — the Audited log is canonical, so the backend choice is just a cache strategy. Add SQLite later by reusing the `MemoryBackend` capability shape.
7. **`SessionId` constructor exported** from `Seal.Core.Types` (changed in M1) so the dispatcher can build a sentinel `"unknown"` session id without a partial `error`. The validator (`mkSessionId`) is still the boundary check.
8. **Postponed milestones:** Tools-Meta (Phase 3 M-b) and Sessions (Phase 3 Sessions group) — user deferred both. Do not pick them up without asking.
